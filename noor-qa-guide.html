1.2 Seed demo data — pick ONE path

Path A — Three SaaS tenants (recommended for subdomain testing)
bash
$ cd wallet-master-api
$ php artisan migrate --force
$ php artisan db:seed --class=TranslationSeeder
$ php artisan db:seed --class=ThreeMadrasasSeeder
$ php artisan storage:link
Path B — Single rich boys madrasa (80 students, full finance)
bash
$ php artisan madrasa:seed-presentation --force
# Login: admin@darululoom.edu / password on localhost (no subdomain required)
Path C — Additive samples only (existing org)
bash
$ php artisan madrasa:seed-essentials

02

Golden rules

Read before every session.

2.1 Subdomain = madrasa

RuleWhy
Always open http://{slug}.localhost:3000 matching your login orgSettings save/load, API tenant header, and branding stay in sync
If redirected after login, follow the redirectApp auto-fixes wrong subdomain
After logout, clear cookies if you switch madrasasAvoids stale tenant_slug / session

2.2 Who can open what

PageMain adminTeam (family)Guardian
/admin/settings?tab=business✅ edit❌❌
/admin/settings profile/security✅✅❌
/admin/settings?tab=localization✅❌❌
/admin/dashboard✅if permitted❌
/parent/*❌❌✅

2.3 Common error messages

MessageUsually meansFix
permission denied (settings.view)Not main admin, or API 403 (see Network tab)Log in as main; use correct subdomain
Forbidden: user does not belong to this tenantWrong subdomain + broken org linkRe-seed ThreeMadrasasSeeder; log in on matching subdomain
Your account is not linked to an active madrasaUser organization_id points to deleted orgRe-run seeder or fix DB link
Save success but refresh shows old nameSaved to different org than URL showsUse {slug}.localhost for your account
