QuickCart

Post-Deploy Full QA Checklist

When to use: After deployment to Dev, Staging, or Production.

Goal: Confirm the site is safe for real traffic and sales — no blank pages, 500s, broken images, or broken checkout flow.

Replace BASE_URL with your deployed URL.
Example: https://dev.yourdomain.com

Related:

  • production-smoke-checklist.md
  • performance-plan.md

0. Deploy Finish (Server)

Run After Deployment

cd /path/to/quickcart

composer install --no-dev --optimize-autoloader
npm ci && npm run build
php artisan migrate --force
php artisan quickcart:ensure-storage

# Only if catalog images are missing:
# php artisan quickcart:ensure-storage --regenerate-missing

php artisan cache:clear
php artisan config:cache
php artisan route:cache
php artisan view:cache

# restart PHP-FPM / queue workers if needed

.env Verification

Setting Expected
APP_ENV production (or staging)
APP_DEBUG false
APP_URL Exact public URL
INERTIA_SSR_ENABLED false unless SSR worker running
CACHE_STORE redis preferred
Storage storage/ and bootstrap/cache writable
Symlink public/storage is symlink

Quick Curl Validation

BASE_URL=https://your-deployed-host

curl -fsS -o /dev/null -w "%{http_code}\n" "$BASE_URL/up"
curl -fsS -o /dev/null -w "%{http_code}\n" "$BASE_URL/"
curl -fsS -o /dev/null -w "%{http_code}\n" "$BASE_URL/products"
  • All responses should return 200
  • If home page returns 500, stop QA immediately
  • Check storage/logs/laravel.log

1. Health & Assets (5 Minutes)

  • GET /up → 200
  • Hard refresh home page
  • No white screen
  • No fatal React/Inertia/usePage errors
  • Favicon and logo load
  • CSS and JS load from /build/assets
  • Images under /storage return 200

Fail Conditions

  • Blank white page
  • 500 on home page
  • Images pointing to localhost

2. Storefront Browsing (10-15 Minutes)

Home

  • Hero/banner slider works
  • Featured/New/Best Seller sections render
  • Category images look correct
  • Search suggestions work
  • Footer links open correctly

Shop (/products)

  • Product grid loads
  • Filters work
  • Pagination works
  • Add to cart updates count
  • No skeleton flash after add-to-cart

Storefront Browsing Continued

Category

  • Parent category URL works
  • Child category URL works
  • Products match category

Product Detail Page (PDP)

  • Open at least 2 products
  • Gallery loads
  • No broken thumbnails
  • Variants update stock and price
  • Add-to-cart works
  • Related products load

Blog (if enabled)

  • /blog loads
  • Post page loads

Fail if PDP/PLP returns 500, gallery missing, or blank Inertia page.

3. Cart & Coupons (10 Minutes)

  • Cart page loads
  • Quantities editable
  • Totals refresh correctly
  • Remove item works
  • Save-for-later works

Coupon Regression Test

  • Apply valid coupon
  • No 500 errors
  • Reload still shows discount
  • No __PHP_Incomplete_Class
  • Remove coupon successfully
  • Apply invalid coupon
  • Proper error shown

Fail if coupon apply/remove causes 500 or Incomplete_Class.

4. Checkout & Order Flow

Checkout Form

  • /checkout loads
  • Validation messages appear
  • Submission does not hang

Order Completion

  • Place COD or sandbox order
  • Confirmation page loads
  • Tracking number generated
  • Order appears in admin
  • Customer notifications work

Tracking

  • /track finds order by tracking number

Fail if checkout 500, blank confirmation, or missing order.

5. Auth & Account

  • Login works
  • Logout works
  • Customer account pages load
  • Guest checkout works (if intended)
  • Admin login reaches dashboard

6. Admin QA (20-25 Minutes)

Dashboard

  • Dashboard loads
  • Hard refresh works

Products (Critical)

  • /admin/products loads
  • Create page loads
  • Publish button visible
  • Tags/categories selectable
  • Create product succeeds
  • Edit product succeeds
  • Media URL starts with /storage

Admin QA Continued

Media

  • Media library loads
  • Upload image succeeds
  • Public URL returns 200

Catalog Extras

  • Categories OK
  • Brands OK
  • Tags OK
  • Coupons editable

Promotions & Settings

  • Banner save works
  • Settings persist
  • Currency/logo/theme correct

Orders

  • Test order visible
  • Status updates work

Fail if create product causes 500, Tag not found, missing toast, or blank admin.

7. Blank Page & Toast Regression

  • Hard refresh home after publish
  • Hard refresh admin after publish
  • Success toasts appear
  • Error toasts appear
  • Add-to-cart toast appears
  • No fatal React errors after multiple navigations

Fail if white screen appears after Inertia navigation.

8. Mobile Responsive QA

  • Home hero usable
  • Filters usable
  • Add-to-cart accessible
  • Cart readable
  • Checkout usable

9. Performance Sanity Check

  • Home loads faster second time
  • Shop browsing feels responsive
  • No multi-second blank screen before Inertia paint

Optional Checks

curl -sI "$BASE_URL/" | head -20
curl -sI "$BASE_URL/api/search-suggestions?q=a" | head -20

10. Log Monitoring

tail -f storage/logs/laravel.log

Must NOT Appear

  • __PHP_Incomplete_Class
  • SeoMetadataCache
  • Class "App\Http\Controllers\Admin\Tag"
  • Permission denied on storage
  • Repeated 500 stack traces

Acceptable for Later Fix

  • Analytics noise
  • Known disabled optional features

11. Sales Readiness Gate

Gate Status
/up and home 200 ☐
Images load from /storage ☐
Add-to-cart works ☐
Coupon apply & remove works ☐
Checkout completed ☐
Admin create/edit product works ☐
APP_DEBUG=false ☐
Logs clean for 10+ minutes ☐
Payment method tested ☐

If any gate fails: DO NOT ANNOUNCE SALES.

12. Recommended QA Flow

  1. Server deploy + /up
  2. Home / Shop / PDP / Images
  3. Cart + Coupon apply/remove
  4. Checkout real order
  5. Admin product create/edit
  6. Settings & promotions
  7. Mobile QA
  8. Log review
  9. Sales readiness review
Estimated Time:
  • Full QA Pass: 60–90 Minutes
  • Hotfix Smoke Test: ~20 Minutes
